Manage Workstreams

Workstreams in Journey Creator are used to configure the UI flow of your existing applications, business features, and steps. You can create multiple workstreams to achieve your desired results, each of which can be easily adjusted and customized from the main App Builder tab. This section describes how to manage your workstreams from the Version sub-tab.

Note: Refer to the Workstreams section for more details about how to configure workstreams.

Complete these steps to manage your workstreams:

  1. In the directory carousel, click the project for which you want to create, edit, or delete a workstream.
  2. In the directory, click the application for which you want to create, edit, or delete a workstream.
  3. Click the App Builder tab, and then click the Version sub-tab.
  4. Click Workstreams, and the Workstreams screen displays:

Note: The name of the parent workstream displays in the Parent column for child workstreams.

  1. To create a new workstream, click New Workstream, and the New Workstream screen displays:

  2. Enter a name for the new workstream into the Name field. The name must be between 5 and 35 characters in length and must contain only letters, numbers, and other approved characters.

    3. Note: Blank spaces are not allowed.

  3. If desired, enter a description into the Description field. The description must be no longer than 150 characters.
  4. At the bottom of the right pane, choose a type for the workstream.
  5. Click Source to choose a source for the new workstream:

    SourceAction

    Workstream

    Select a parent workstream from the drop-down list to create child workstream.

    Note: When you delete a parent workstream, the child workstream is automatically deleted.

    Import File

    Click Select File to upload a file.

    Git Pull

    Enter the repository and branch.

  6. Click Save.
  7. To edit an existing workstream, click the desired workstream from the list, and the Edit Workstream screen displays:

    • To edit the name, description, or type of the workstream, click Edit, modify the information as desired, and then click Save.
    • To back up your workstream, click Export, and select Export File to export your backup to a file, or select GIT push to push the backup to GIT, and then click Export.
  8. To delete a workstream, click the checkbox next to the desired workstream in the list. Click Delete, and when the confirmation message displays, click Yes.

Note: When you delete a parent workstream, the child workstream is automatically deleted.
